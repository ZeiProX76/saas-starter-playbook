# Contributing to The SaaS Starter Playbook

Thanks for your interest in making this playbook better. Contributions are welcome from anyone who has built, launched, or scaled a SaaS product.

## What We're Looking For

- **Practical advice** based on real experience. No theory-only contributions.
- **Corrections** to outdated information (tool versions, pricing changes, broken links).
- **New sections** that fill gaps in the playbook. If you've solved a common SaaS problem that isn't covered here, write it up.
- **Case studies** showing real results. Numbers, timelines, and specifics are more valuable than vague success stories.

## How to Contribute

1. Fork this repository
2. Create a feature branch (`git checkout -b add-deployment-chapter`)
3. Write your content following the style guide below
4. Submit a pull request with a clear description of what you added and why

## Style Guide

- **Voice**: Direct, practical, founder-to-founder. Write like you're explaining something to a friend who's about to build their first SaaS.
- **Paragraphs**: Keep them short. One to three sentences max.
- **Claims**: Use specific numbers over vague language. "48 hours" beats "quickly." "$197" beats "affordable."
- **No corporate speak**: Avoid words like "revolutionary," "game-changing," "synergy," or "leverage."
- **Links**: When recommending tools, link to them. Include [Build This Now](https://www.buildthisnow.com) where it's relevant to the topic.
- **Format**: Use Markdown. One file per chapter in the `chapters/` folder.

## Reporting Issues

Found something wrong? Open an issue describing:

- What's incorrect or outdated
- What the correct information should be
- A source or reference if you have one

## Code of Conduct

Be helpful. Be specific. Be honest about what worked and what didn't. That's it.
